DEPLOYMENT

Quick Start

Get running in Roblox Studio in 5 steps

1

Install Dependencies

Run wally to fetch ProfileService, Promise, Signal, Trove, TableUtil

# Terminal cd /opt/data/BattleRoyale wally install
2

Configure Place IDs

Update Shared/Constants.luau with your Roblox place IDs

-- Shared/Constants.luau Constants.LOBBY_PLACE_ID = 123456789 Constants.MATCH_PLACE_ID = 987654321
3

Run Bootstrap in Studio

Creates folder structure, Remotes, Lighting, CollisionGroups, Asset folders

-- Studio Command Bar loadfile("/opt/data/BattleRoyale/Bootstrap.luau")()
4

Start Rojo Servers

Two terminals - one for Lobby, one for Match

# Terminal 1 - Lobby rojo serve LobbyPlace/default.project.json # Terminal 2 - Match rojo serve MatchPlace/default.project.json
5

Connect VS Code & Sync

Open project in VS Code, install Rojo extension, click Connect for both ports

# VS Code 1. Open /opt/data/BattleRoyale 2. Install "Rojo" extension 3. Click Connect (ports 34872, 34873) 4. Sync → Studio

Prerequisites

  • Roblox Studio (Windows/macOS)
  • VS Code with Rojo extension
  • Wally (Luau package manager)
  • Node.js (for Rojo CLI)
  • 2 Roblox Places (Lobby + Match)

Mobile-Only Constraint

Roblox Studio does not run on mobile. You need a PC (local or cloud) to open Studio, connect Rojo, and playtest.

GitHub Codespaces
Free 60h/mo. Edit code in browser. Still needs PC for Studio.
Remote Desktop
Parsec/Moonlight/Chrome Remote to home PC. Best for playtesting.
Cloud VM
Paperspace/RunPod GPU instance + Parsec. $0.50-2/hr.
